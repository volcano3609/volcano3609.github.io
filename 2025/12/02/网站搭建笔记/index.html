网站搭建笔记

我是 CodeArchitect。
这份文档不仅是一份教程，更是你从“代码小白”到“拥有独立网站”的全过程实战复盘。我将我们刚才经历的所有步骤、遇到的每一个报错坑、以及解决思路，都整理成了这份**《零基础 Hexo + GitHub Pages 个人博客搭建全攻略》**。
建议收藏，或者打印下来，作为你日后维护博客的手册。
📘 零基础 Hexo + GitHub Pages 个人博客搭建全攻略
作者：CodeArchitect & Volcano3609
适用人群：完全不懂代码的初学者
核心目标：免费搭建一个高逼格、Notion 风格的个人网站，并托管在 GitHub 上。
📅 第一阶段：环境准备 (打地基)
在开始盖房子之前，我们需要准备好工具。

  1. 安装核心软件
    Node.js (Hexo 的运行引擎)：
    下载官网 LTS 版本。
    踩坑记录：如果版本太旧（如 v20.14），会导致后续报错 EBADENGINE。
    解决方案：务必安装最新 LTS 版本（v20.19+ 或 v22+）。
    Git (搬运工)：
    去官网下载 Git for Windows，一路 Next 安装。
  2. 注册 GitHub 仓库 (圈地)
    仓库名规则（死命令）：必须是 你的用户名.github.io。
    例如：你的用户名是 volcano3609，仓库名必须是 volcano3609.github.io。
    设置：选择 Public (公开)。
    🛠️ 第二阶段：本地搭建 (盖房子)
    我们将使用 Hexo 框架在你的电脑上生成博客文件。
  3. 初始化 Hexo
    在你的电脑上新建文件夹（如 My website），右键打开 Git Bash。
    执行命令：
    code
    Bash
    npx hexo init
    为什么用 npx？：
    小白痛点：直接输 hexo 会报错 command not found，因为环境变量没配好。
    架构师解法：npx 是 Node.js 自带的神器，它能自动找到 hexo 运行，免去配置环境变量的烦恼。
  4. 安装依赖
    初始化完成后，执行：
    code
    Bash
    npm install
    这步是安装 Hexo 需要的各种插件（相当于买家具）。
  5. 本地预览
    code
    Bash
    npx hexo s
    打开浏览器访问 http://localhost:4000。如果有画面，说明本地搭建成功。
    🚀 第三阶段：发布上线 (通水电)
    要把电脑里的博客推送到 GitHub 让全世界看到。
  6. 获取 GitHub Token (钥匙)
    GitHub 网页 -> Settings -> Developer settings -> Personal access tokens (Classic)。
    权限：勾选 repo 这一项。
    保存：复制那个 ghp_ 开头的字符串，只显示一次！
  7. 配置 _config.yml (身份证)
    用记事本打开根目录下的 _config.yml，修改底部的 deploy 部分：
    code
    Yaml
    deploy:
    type: git
    repo: https://github.com/volcano3609/volcano3609.github.io.git
    branch: main
  8. 安装部署插件
    在 Git Bash 执行：
    code
    Bash
    npm install hexo-deployer-git –save
  9. 首次部署与身份验证 (最容易报错的一步)
    执行发布命令：
    code
    Bash
    npx hexo g -d
    🛑 常见报错与解决：
    报错：Author identity unknown
    原因：Git 不知道你是谁。
    解决：
    code
    Bash
    git config –global user.name “你的GitHub用户名”
    git config –global user.email “你的注册邮箱”
    报错：Recv failure: Connection was reset
    原因：网络问题，连接 GitHub 失败。
    解决：
    多试几次。
    或者设置 Git 缓存：git config –global http.postBuffer 524288000。
    实在不行用手机热点。
    🎨 第四阶段：装修主页 (Notion 风格)
    Hexo 默认的主页是文章列表，我们要把它改成高大上的个人主页。
  10. 腾出首页位置
    Hexo 默认霸占了首页。我们需要把文章列表赶到 /blog 目录下。
    操作：修改 _config.yml，添加或修改以下内容：
    code
    Yaml
    index_generator:
    path: blog
    per_page: 10
    order_by: -date
  11. 植入自定义主页
    操作：在 source 文件夹下，新建 index.html。
    内容：复制我们设计好的 HTML 代码（Notion 风格、双栏布局）。
    关键点：文件头必须包含 layout: false，防止 Hexo 破坏我们的设计。
  12. 应用更改
    因为修改了核心配置，必须执行**“三连击”**：
    code
    Bash
    npx hexo clean # 清除缓存 (这一步不做，配置可能不生效)
    npx hexo g # 重新生成
    npx hexo d # 发布
    ✍️ 第五阶段：日常写作与维护 (入住)
    这是你以后每天要重复的工作流。
  13. 新建文章
    code
    Bash
    npx hexo new “文章标题”
    或者直接复制 source/_posts/ 里的 .md 文件并重命名。
  14. 设置文章分类 (关键)
    打开 .md 文件，在顶部的“文章头”里加上 categories：
    code
    Yaml

title: 网站搭建笔记
date: 2025-12-02 10:00:00
categories: 网站搭建 <– 这一行决定了分类

  1. 获取分类链接
    Hexo 会自动生成分类页面，链接规律是固定的：
    公式：/categories/你的分类名/
    例如：/categories/网站搭建/
  2. 修改主页导航链接
    用记事本打开 source/index.html，找到对应的按钮，把 href 改成上面的链接：
    code
    Html

❓ 常见问题急救手册 (FAQ)
Q1: 我发布成功了 (Deploy done)，但网页没变化？
原因 A：浏览器缓存。
解决：按 Ctrl + F5 强制刷新，或者用手机流量访问。
原因 B：GitHub 服务器延迟。
解决：等 2-5 分钟。
Q2: 点击左侧导航栏，显示 404？
原因：你写了分类，但还没发布；或者链接填错了。
解决：
确保 .md 文件里写了 categories: xxx。
确保执行了 npx hexo g -d。
链接地址必须是 /categories/xxx/ (注意前后斜杠)。
Q3: 记事本修改代码要注意什么？
后缀名陷阱：确保文件叫 index.html，而不是 index.html.txt。请在文件夹视图里开启“显示文件扩展名”。
编码：记事本默认是 UTF-8，通常没问题。
